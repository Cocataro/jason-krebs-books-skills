# Launch T-Minus Runbook

Per-book schedule from manuscript lock to launch day.

## T-60 (manuscript locked by Line/Copy Editor + EIC ship-signed-off)
- Blurb & Metadata Writer: start blurb v1 draft from final text
- KDP Ops: start formatting work in Vellum. Build template if Book 1; update template for Books 2+.
- KDP Ops: notify Cover Art Director that manuscript is final and interior work has started.

## T-55
- Blurb shared with EIC for voice/alignment review

## T-50
- Blurb revised with EIC notes, shared with CEO Jason for brand-level sign-off (title, subtitle, author bio, series name confirmation)

## T-45
- Cover final for ebook — Cover Art Director delivers ebook JPG (2560×1600) to KDP Ops
- Subtitle + title lockup confirmed with Cover Art Director
- Blurb locked

## T-40
- Interior final from Line/Copy Editor → format in Vellum
- Kindle Previewer pass on export
- EPUBCheck validation on EPUB

## T-35
- Paperback interior final — page count locked
- KDP Ops sends page count to Cover Art Director for spine-width calculation

## T-32
- Paperback cover PDF received from Cover Art Director
- KDP Ops validates spine width against page count × 0.0025" (cream)
- PDF preflight: embedded fonts, bleed/trim, no errors

## T-30
- **Upload Packet complete.** KDP Ops hands to Nicole.
- CEO Jason approves pre-order open date + launch pricing + KU enrollment confirmation.
- Keyword research complete in Publisher Rocket; 7 phrases finalized.
- Category selection complete; 3 paths confirmed against current Amazon tree.
- A+ Content copy finalized; imagery ready from Cover Art Director.

## T-30 to T-7 (Nicole's upload window)
- Nicole uploads ebook (pre-order opens T-30 for Books 2–6; Book 1 has NO pre-order so uploads at T-7 for same-day launch).
- Nicole uploads paperback interior + cover; orders proof (arrives T-14–T-20).
- KDP Ops monitors dashboard for acceptance. Handles any KDP support tickets.

## T-14
- Author Central packet executed by Nicole (bio, photo, RSS, Follow setup, ASIN claim)
- Series Page created or verified (Book 2+)

## T-7
- Final pre-launch audit: every field verified via product-page scrape
- Paperback proof approved by Nicole (for paperback launch)
- Any corrections made
- ARCs distributed (when Marketing is hired; until then, skip or Board runs ARC plan directly)

## T-0 Launch Day
- Book goes live (Book 1 hard launch; Books 2–6 pre-order delivers)
- KDP Ops monitors: rank hourly, KENP read, reviews, category placement
- Ads turn on (when Marketing is hired)
- Social reveal assets deployed (when Marketing is hired)
- CEO Jason gets launch dashboard

## T+7 (launch week close)
- Book 1: $0.99 Countdown Deal ends; price settles to $4.99
- KDP Ops posts launch-week report: units sold, KENP read, ranks, reviews, any anomalies

## T+14
- Paperback proof accepted (if not already) → paperback goes live

## T+30
- First metadata-refresh signal: based on 30-day conversion, Blurb & Metadata Writer may refresh blurb, keywords, categories
- KDP Ops hands updated packet to Nicole if refresh triggered
- CEO gets 30-day performance report

## Cadence across the 6-book slate

- Rapid-release cadence target: one book every 6–8 weeks during launch window. Matches KU cliff algorithm.
- When Book N-1 launches, Book N is in T-30 window, Book N+1 is in T-60 window.
- Stagger: EIC + Dev Editor + Line/Copy Editor must schedule per-book so no two books hit T-30 simultaneously.
