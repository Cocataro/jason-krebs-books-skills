# Preserve-vs-Query Rule

The single most important decision in every line + copy pass.

## The Rule

> **PRESERVE by default when the passage shows intent markers** (recurrence, rhythm function, dialect marker, style-sheet entry). **EDIT only when** (a) genuine reader confusion, (b) continuity break, or (c) explicit style-sheet rule violation. **When uncertain, QUERY in a tracked comment — never silent-correct.**

CMOS is the fallback. The style guide's humanizing-fiction directive is the master.

## Intent Markers (signals to preserve)

### 1. Pattern Recurrence
- Fragment once on page 47 = probably error
- Fragment every 3–5 paragraphs in this POV = **voice**
- Run-on once = probably bloat
- Run-on only in Briar's anxiety beats = **voice**

### 2. Contextual Function
- A fragment at an emotional punch = earning its keep
- A run-on during a breathless sequence = earning its keep
- A sentence fragment in the middle of straightforward exposition = probably error

### 3. Style-Sheet Precedent
- Already codified in the style sheet? Preserve.
- Query if uncertain whether the style-sheet entry applies to this specific case.

### 4. Authorial Signal
- Series Style Guide declares this tic = preserve.
- Character bible says this character says "honest" as filler = preserve every instance.

### 5. Dialect Marker
- Regional idiom documented in style sheet = preserve.
- Character known to drop "g" on -ing words = preserve.

## When to Edit (the only times silent-correction is OK)

- **Genuine error** — obvious typo, missing word, wrong homophone (their/there, its/it's confusion), dropped pronoun, subject-verb disagreement that isn't dialect.
- **Continuity break** — eye color changed, name spelled two ways, timeline contradicts prior book.
- **Explicit style-sheet rule violation** — invented term spelled wrong per style sheet; formatting convention broken; dialect mismatched to character.

## When to Query (not edit)

- Anything that LOOKS like error but might be intent.
- Anything you'd normalize on autopilot — stop, query instead.
- Humanizing-fiction fingerprints (fragments, run-ons, uneven paragraphs, regional idiom) you're not 100% sure are codified.
- CMOS-says-X but the passage reads right — query, let the author/EIC rule.

## Query Format

Margin comment, starts with `[Q]`:

```
[Q] Fragment — intentional? (pattern matches pp. 12, 34, 61 — may want to codify in style sheet)
```

## Never

- **Silent-correct a potentially-intentional imperfection.** Terminating offense.
- **Query a clear error.** Just fix it. Too many queries on clear errors = timid editor.
- **Quote CMOS at the author.** CMOS is a resource; the book is the master.
- **Normalize paragraph-length chaos.** That's the humanizing-fiction mandate.

## Counter-Examples

### Example 1: Silent-correct is CORRECT
> Original: "She walked threw the door."
> Edit: "She walked through the door."

Clear homophone error. Not intent.

### Example 2: Silent-correct is WRONG
> Original: "Proper cold out. Proper cold."
> Edit: "It was proper cold out. The air was proper cold."

Wrong — "proper" as adverb is a documented regional dialect marker. The original fragment pattern is voice. Should have been preserved.

### Example 3: Query is CORRECT
> Original: "And then she saw it. The cup. Broken again."
> Query: `[Q] Fragments for impact — intentional pattern? Recurrences on pp. 14, 89. Consider codifying in style sheet if house.`

Preserve with a note, let EIC codify.

### Example 4: Edit is CORRECT
> Original: "The Inn's hearth glowed amber."
> Cross-check: Earlier chapter says "emerald-green hearthlight."
> Edit + note: Continuity break — made consistent with earlier; `[Q] confirm: canonical hearth color is amber or emerald-green? Multiple chapters affected.`

Continuity break. Fix + query for ruling.
